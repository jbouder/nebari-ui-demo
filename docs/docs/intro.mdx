# Introduction

Welcome to the **Nebari UI** documentation — the home for docs and live demos of
the [Nebari design system](https://github.com/nebari-dev/nebari-design).

These docs sit alongside the `nebari-ui-demo` application and are built with
[Docusaurus](https://docusaurus.io/). They serve two purposes:

- **Document** how the Nebari design system is consumed in a real React + Vite app.
- **Demo** the components live, so you can see and interact with them right here
  in the documentation.

## What is the Nebari design system?

The Nebari design system is distributed as a [shadcn](https://ui.shadcn.com/)
component registry. Components are added to an app on demand with the registry
configured in `components.json`:

```bash
npx shadcn@latest add @nebari/button
npx shadcn@latest add @nebari/spinner
npx shadcn@latest add @nebari/theme
```

Under the hood the components are built on:

- **React 19** + **TypeScript**
- **Tailwind CSS v4** with design tokens in the `oklch` color space
- **Base UI** primitives for accessible, unstyled behavior
- A brand **purple** primary color with first-class **light and dark** modes
